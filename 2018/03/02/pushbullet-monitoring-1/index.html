Pushbullet 을 이용하여 서버 모니터링 하기-1

2018-03-02
Monitoring, Push alert, Pushbullet

서버 모니터링을 하며, 해당 모니터링중 서버로부터 통신이 되지 않을때 해당 사항을 스마트폰 Push Alert 을 받을 수 있는 방법에 대해서 알아보겠습니다. 우선은 Push Alert을 받을 수 있도록 해주는 Pushbullet 이라는 서비스에 대해서 알아보겠습니다.

1. Pushbullet 이란

Pushbullet은 각 디바이스(PC브라우저, 스마트폰 등)별로 파일 및 문자를 전송시킬 수 있으며 이를 동기화시켜 줍니다. 예를 들어 PC브라우저와 스마트폰에서 메시지를 전송하면 동기화 되어있는 디바이스(브라우저 또는 스마트폰)에서 Push 알람이 울리게 도와줍니다.

2. Pushbullet 가입하기

우선은 Pushbullet 홈페이지에 접속후 Sign in버튼을 클릭합니다.

이후 Facebook 또는 Google 계정을 가입을 진행하시면 됩니다. (저는 Facebook 계정으로 가입을 진행하였습니다.)

가입을 진행 완료하시면 아래와 같이 계정 관리 화면이 나오게 됩니다.

관리 화면에서 동기화된 기기를 관리할 수 있으며, 친구를 등록하는 등 설정이 가능합니다. 왼쪽 Setup 메뉴에서는 친절하게 사용자의 디바이스별로 동기화 할 수 있는 방법에 대해서 안내합니다.

안내하는 방법에 따라서 스마트폰(아이폰) 과 브라우저(크롬) 를 동기화 시키는 작업을 진행하도록 하겠습니다.

우선 자신의 스마트폰에 맞는 어플을 설치 합니다. 저는 아이폰이라서 앱스토어에서 Pushbullet 앱을 다운 받았습니다.

이후 브라우저(크롬)에 플러그인 설치하는 방법인데, 크롬 확장기능으로 Pushbullet 확장 플러그인이 있습니다. 이를 설치 합니다.

크롬 웹 스토어에서 pushbullet 을 설치합니다.

이후 스마트폰에서 채팅 메시지를 보내면 아래와 같이 브라우저에서 알람이 오는것을 확인 할 수 있습니다.

마찬가지로 브라우저에서 메시지를 발송하였을 때 스마트폰에서 Push 알람이 울린다면 서로 동기화가 성공적으로 이루어 졌다고 볼 수 있습니다.

다음 글에서 Access Token 을 발급받고 서버 에서 직접 Push 메시지 를 보내는 방법을 알아보도록 하겠습니다.

감사합니다.
